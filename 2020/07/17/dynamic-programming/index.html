算法思维：动态规划

动态规划（Dynamic programming，DP）把原问题分解为子问题求解复杂问题，其适用于有重叠子问题和最优子结构性质的问题，所耗时间往往远少于朴素解法（空间换时间）。

  1. 最优子结构：如果问题的最优解中包含了子问题的最优解，则该问题具有最优子机构。
  2. 重叠子问题：用于原问题求解的递归算法，反复地解同样的子问题，而非产生新的子问题。

动态规划法步骤如下：

  1. 描述最优解的结构；
  2. 递归定义最优解的值，即递推公式；
  3. 自底向上计算子问题的最优解，并记录，便于查表。

动态规划问题的常见形式就是求最值，例如最长递增子序列、最小编辑距离等。

一. 凑零钱

Leetcode 322. Coin Change：
$k$种面值的硬币，面值分别为$c_1, c_2 … c_k$，问最少需要几枚硬币凑出金额amount，若无法凑出，返回-1。

当然，使用基于回朔算法的穷举算法可以解决问题，但动态规划可获得更优解，递推公式如下: $$\text{dp}(n) = \min \left\{\text{dp}(n-\operatorname{coin})+1 \mid \operatorname{coin} \in \operatorname{coin} s\right\}, n>0$$

注$\star$：求最小值，dp table 通常初始化为极大值，反之亦然。

Leetcode 518. Coin Change 2：
$k$种面值的硬币，面值分别为$c_1, c_2 … c_k$，问有几种方式可以凑出金额amount，若无法凑出，返回0。

回朔算法可行，动态规划可优化，重复子问题的求解。采用完全背包问题（物品数量无限制）的思想，定义$\text{dp}[i][j]$：使用前$i$类币，凑出金额$j$的方式数。若不使用第$i$类币，$\text{dp}[i][j]=\text{dp}[i-1][j]$；若使用第$i$类币，拿出一个$i$类币，$\text{dp}[i][j] = \text{dp}[i][j-coins[i]]$。故其递推公式为：$$\text{dp}[i][j] = \text{dp}[i-1][j] + \text{dp}[i][j-coins[i]])$$。

二. 切钢条问题

不同长度$l_i$的钢条的售价为$v_i$，问长度为$n$的钢条最高售价。

同上，也是完全背包问题，可以用一个二维DP解决，定义$\text{dp}[i][j]$：使用前$i$类切割，长度为$j$的钢条最高售价。递推公式为：$$\text{dp}[i][j] = \text{max}(\text{dp}[i-1][j], \text{dp}[i][j-l[i]])$$

1维DP，定义$\text{dp}[i]$：度为$i$的钢条最高售价，$$\text{dp}[i] = \text{max}(\text{dp}[i-l_j] + \text{vals}[j]\mid l_j \in l)$$

三. 背包问题

Leetcode 416. Partition Equal Subset Sum：
背包最大载重为$W$，$N$个物品，其中第$i$个物品的重量为$\text{w}_i$，价值为$\text{val}_i$，问起最多能装的价值是多少？

定义$\text{dp}[i][w]$：对于前$i$个物品，当背包的容量为$w$时，可以装载的最大价值。若不装第$i$个物品，$\text{dp}[i][w]=\text{dp}[i-1][w]$；若装第$i$个物品，$\text{dp}[i][w] = \text{dp}[i-1][w-ws[i]] +vals[i]$。故其递推公式为：$$\text{dp}[i][w] = max(\text{dp}[i-1][w], \text{dp}[i-1][w-ws[i]] + val[i])$$

注$\star$：1. 背包问题的形式为，元素$i$有价值$v_i$与重量$w_i$，求符合“重量和约束”或“价值和约束”下的的最优值；
注$\star$：2. 背包问题主要就是围绕，前$i$个元素，设计$\text{dp}[i]$；再通过对第$i$个元素的选择，递推$\text{dp}[i]$。

四. 股票问题

1. 一次股票交易：

121. 买卖股票的最佳时机

2. 不限次股票交易：

122. 买卖股票的最佳时机 II

使用2d-DP解决，定义：$\text{dp}[i][1]$为第$i$日，持有股票时的最大收益；$\text{dp}[i][1]$为第$i$日，未持有股票时的最大收益；最终结果为$\text{dp}[N][0]$。

第$i$日，持有股票时，最大收益$\text{dp}[i][1]$为：若第$i$日，没有买入股票，则$i-1$日就持有股票，故$\text{dp}[i][1] = \text{dp}[i-1][1]$；若第$i$日，买入股票，则$i-1$日必然未持有股票，故$\text{dp}[i][1] = \text{dp}[i-1][0] - \text{prices}[i]$；那么，综合可知道，第$i$日持有股票时的递推公式：$$\text{dp}[i][1] = \text{max}(\text{dp}[i-1][0] - \text{prices}[i], \text{dp}[i-1][1])$$同理，第$i$日，未持有股票时递推公式为：$$\text{dp}[i][0] = \text{max}(\text{dp}[i-1][1] + \text{prices}[i], \text{dp}[i-1][0])$$

初始化$\star$：第0天，若持有股票，收益为-inf；第0天，若持不持有股票，收益为0。

进一步简化，统计所有$\text{prices}_i - \text{prices}_{i-1} > 0$：

309. 最佳买卖股票时机含冷冻期

卖出之后，需等待一天再交易。故第$i$日持有股票，若其为第$i$日买入，那么第$i-2$日，必不持有股票。递推公式为：

$$\text{dp}[i][0] = \text{max}(\text{dp}[i-1][1] + \text{prices}[i], \text{dp}[i-1][0])\\ \text{dp}[i][1] = \text{max}(\text{dp}[i-2][0] - \text{prices}[i], \text{dp}[i-1][1]) $$

714. 买卖股票的最佳时机含手续费

$$\text{dp}[i][0] = \text{max}(\text{dp}[i-1][1] + \text{prices}[i] - \text{fee}, \text{dp}[i-1][0])\\ \text{dp}[i][1] = \text{max}(\text{dp}[i-1][0] - \text{prices}[i], \text{dp}[i-1][1]) $$

3. 有限次数股票交易：

123. 买卖股票的最佳时机 III、188. 买卖股票的最佳时机 IV

此时，需要增加一维状态，用以表示交易次数（每买入一次，进入一笔新交易），需要用到3d-DP。递推公式为：

$$\text{dp}[i][k][0] = \text{max}(\text{dp}[i-1][k][1] + \text{prices}[i], \text{dp}[i-1][k][0])\\ \text{dp}[i][k][1] = \text{max}(\text{dp}[i-1][k-1][0] - \text{prices}[i], \text{dp}[i-1][k][1]) $$

注$\star$：和前面保持一样，第0日，持有股票时的收益为-inf。

六. 最小编辑距离/最长子序列

Leetcode 72. 编辑距离，变体

定义$\text{dp}[i][j]$：word1前$i$个字符，转换至word2前$j$个字符，所需要的最小编辑距离。递推公式为：$$\text{dp}[i][j] = \text{dp}[i-1][j-1]\quad \text{if } \text{word}[i] = \text{word}[j] \\ \text{dp}[i][j] = \text{min}(\text{dp}[i-1][j], \text{dp}[i-1][j-1], \text{dp}[i][j-1]) + 1\quad \text{if } \text{word}[i] \neq \text{word}[j] $$
